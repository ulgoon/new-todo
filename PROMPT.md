React와 Tailwind CSS로 할 일(TODO) 관리 웹 앱을 만들어줘. 아래 요구사항을 모두 반영해줘.

1. 화면 레이아웃
- 할 일이 하나도 없을 때: 화면 정중앙에 텍스트 입력창과 추가 버튼을 배치한다.
- 할 일이 1개 이상 있을 때: 정중앙 입력창은 사라지고, 할 일들이 Card UI 그리드로 나열된다. 데스크톱에서는 4열, 태블릿에서는 2열, 모바일에서는 1열로 배치되는 반응형 그리드로 구성한다. 각 카드에는 할 일 이름이 표시된다. 마지막 항목을 삭제해 목록이 다시 비면 정중앙 입력창 상태로 돌아간다.

2. 할 일 추가
- 목록이 비어 있을 때는 정중앙 입력창에 내용을 입력한 뒤 추가 버튼 클릭 또는 [Shift + Enter]로 추가한다.
- 목록에 항목이 있을 때는 화면 우측 하단의 플로팅 버튼(FAB)을 클릭하면 입력용 모달 창이 열린다. 모달의 입력창에 내용을 입력한 뒤 추가 버튼 클릭 또는 [Shift + Enter]로 추가하면 모달이 닫힌다. 모달은 배경 오버레이와 함께 화면 중앙에 뜨고, 바깥 영역 클릭 또는 닫기 버튼으로도 닫을 수 있다.
- 두 경우 모두 입력값이 비어 있으면 추가되지 않는다.

3. 할 일 상태 관리 (각 카드에 버튼으로 표현)
- 모든 할 일은 '진행중', '완료', '취소' 중 하나의 상태를 가지며, 새로 추가되면 '진행중'으로 시작한다.
- 완료: '진행중' 카드의 '완료' 버튼을 누르면 상태가 '완료'로 바뀌고, 완료 처리 시각이 함께 기록되어 카드에 표시된다.
- 취소: '완료' 카드의 '취소' 버튼을 누르면 상태가 '취소'로 바뀌고, 취소 처리 시각이 함께 기록되어 카드에 표시된다. 이미 기록된 완료 시각은 지우지 않고 취소 시각과 함께 보여준다.
- 삭제: 각 카드의 삭제 버튼을 누르면 해당 할 일 데이터가 완전히 제거된다(복구 불가).
- 상태별로 카드를 시각적으로 구분한다(예: 완료/취소 항목은 흐리게 표시하거나 취소선 적용).
- 완료/취소/삭제 버튼은 텍스트 대신 Lucide 아이콘으로 표시하고, 마우스 호버 시 툴팁으로 버튼 이름(완료/취소/삭제)을 보여준다.

4. 오래된 항목 숨김 / 모두 보기
- '완료' 또는 '취소' 상태가 된 지 24시간이 지난 항목은 기본적으로 목록에서 숨긴다(데이터는 삭제되지 않고 유지된다). '진행중' 항목은 항상 표시한다.
- 화면 우측 상단에 '모두 보기' 토글 버튼을 두어, 켜면 24시간이 지나 숨겨진 완료/취소 항목까지 모두 표시하고, 끄면 다시 최근 24시간 이내 항목만 보여준다. 현재 켜짐/꺼짐 상태를 시각적으로 구분해서 보여준다.

5. 정렬 및 필터
- 정렬: '최근순'(생성 시각 최신순), '오래된순'(생성 시각 오래된순), '할 일 제목순' 중 선택할 수 있는 정렬 옵션을 목록 상단에 둔다.
- 필터: '전체', '진행중', '완료', '취소' 중 선택할 수 있는 필터 옵션을 목록 상단에 둔다. 선택한 상태의 할 일만 표시한다.
- 정렬/필터 옵션과 FAB, '모두 보기' 버튼도 Lucide 아이콘으로 표시하고, 마우스 호버 시 툴팁으로 각 항목의 이름을 보여준다.

6. 할 일 수정 모달
- 카드를 클릭하면 해당 할 일을 수정할 수 있는 모달 창이 열린다.
- 모달에는 다음 항목이 포함된다.
  - 할 일 내용: 수정 가능한 텍스트 입력창
  - 마감시간: 수정 가능한 날짜/시간 선택 필드
  - 완료 상태 체크박스: 체크하면 완료 처리되고, 체크 해제하면 완료/취소 상태가 해제되어 '진행중'으로 돌아간다.
  - 위치 입력 필드와, 입력된 위치를 보여주는 지도. 위치 정보가 없으면 지도는 표시하지 않는다.
- 모달의 저장 버튼을 누르면 변경 사항이 반영되고 모달이 닫힌다.

7. 구글 시트 연동 (Google Apps Script 웹앱 방식)
- 할 일 데이터를 Google Apps Script 웹앱을 통해 구글 시트에 저장하고 불러온다. 시트의 컬럼은 `id, title, status, createdAt, dueDate, completedAt, cancelledAt, locationAddress, locationLat, locationLng, updatedAt` 순서로 구성되어 있다.
- Apps Script 웹앱 URL은 환경변수(예: `VITE_APPS_SCRIPT_URL`)로 주입받아 사용한다.
- 앱 최초 로딩 시 웹앱에 GET 요청을 보내 전체 할 일 목록을 받아와 화면에 표시한다.
- 할 일 추가/수정(내용, 마감시간, 상태, 위치 등)/삭제가 일어날 때마다 웹앱에 POST 요청을 보내 시트에 반영한다. 요청 본문은 `{ action: 'create' | 'update' | 'delete', todo?: {...}, id?: string }` 형태의 JSON으로 보낸다.
- Apps Script 웹앱은 브라우저의 CORS 프리플라이트를 피하기 위해 POST 요청의 Content-Type을 `text/plain;charset=utf-8`로 보내고, 본문은 JSON 문자열로 직렬화한다.
- 네트워크 요청 중에는 로딩 상태를 표시하고, 요청이 실패하면 사용자에게 알리고 이전 상태를 유지한다. 성공 시까지는 localStorage에도 동일한 데이터를 캐시해 새로고침 시 빠르게 표시한다.

8. 기술 요구사항
- React와 Tailwind CSS로 구현한다.
- UI 스타일은 shadcn/ui 컴포넌트(Button, Card, Dialog, Select, Checkbox 등)를 활용한다.
- 아이콘은 Lucide 아이콘(lucide-react)을 사용한다.
- 지도 표시에는 OpenStreetMap을 사용한다(예: Leaflet + react-leaflet과 OpenStreetMap 타일 서버 조합).
- 할 일 데이터는 id, 이름, 생성 시각, 상태('진행중'/'완료'/'취소'), 완료 시각, 취소 시각, 마감시간, 위치 정보, 마지막 수정 시각을 포함한 객체로 관리한다.
- 할 일 목록은 Google Apps Script 웹앱을 통해 구글 시트와 동기화하고, localStorage에도 캐시해 새로고침 시 유지되도록 한다.
- 모바일에서도 잘 보이도록 반응형으로 만든다.
